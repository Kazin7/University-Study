컨테이너
• 프로그램을 어느 환경에서나 바로 실행할 수 있도록 규격화된
기술임.
• 이미지를 실행한 상태이며 격리된 공간에서 수행
• 컨테이너는 종료되어도 메모리에 존재하며, 다시 시작 가능함
• 한 서버에서 여러 개의 컨테이너 수행이 가능하며 각각 독립적
으로 수행됨

docker images
내 컴퓨터에 다운로드된 이미지 목록 보기

docker search [이미지명]

docker ps
컨테이너 목록 보기

docker run
• docker run (<옵션>) <이미지 식별자> (<명령어>) (<인자>)
• -d : 백그라운드 실행
• -it : 컨테이너로 터미널 입력 전달
• -name : 컨테이너 이름 부여
• -v : 볼륨 설정

docker start

docker exec

docker attach
컨테이너 터미널 부착

docker kill
실행중인 컨테이너 종료

wordpress
장점
• 다양한 기능의 플러그인과 테마 존재
• 전세계 거의 모든 국가에 사용자 존재, 워드프레스 기반 프리랜서 용이
• 웹사이트를 완전히 커스터마이징할 수 있는 자유도
• 차단율 99%의 스팸 트랙백 차단 엔진 Akismet 사용
• 단점
• WordPress의 사용법을 배우는 데에는 시간이 걸림
• 웹을 완전히 커스터마이징하려면 프로그래밍 언어에 대한 지식이 필요
• 설치형과 서비스형으로 제공되며, 서비스형은 비용이 필요함
• 보안적인 취약점
sudo apt update
sudo apt install nginx
sudo apt install docker.io
sudo apt install docker-compose
mkdir wp
cd wp
nano docker-compose.yml
sudo docker-compose up -d

tmux 구성 요소
session: 여러 윈도우로 구성
window: 터미널 화면, 세션 내에서 탭처럼 사용 가능
pane: 하나의 윈도우 내에서의 화면 분할

기본 : ctrl + b
컨트롤 키와 b 키를 누른다.

pane 나누기 : 
(ctrl + b) % #좌우로 나누기
(ctrl + b) " #위아래로 나누기

pane 이동하기
(ctrl + b) 방향키, 
(ctrl + b) q, 
(ctrl + b) o #순서대로 이동

pane 삭제 : (ctrl + d)

단축키 목록 : (ctrl + b) ?

git add . 명령어는 현재 디렉토리에 있는 모든 파일을 stage 단계에 올려줌
git commit –m “message”는 stage 단계에 있는 파일들에 대해 커밋 생성(새로운 버전 할당)
git branch –M main : 새로운 브랜치 main 생성
git push origin main : main 브랜치를 원격 저장소에 전송

git pull
git status : 파일 상태 확인
